🔗 Linked Data Principles (2006)

Tim Berners-Lee defined four core principles:

  • Use URIs (Uniform Resource Identifiers) as names for things
  • Use HTTP URIs so that people can look up those names
  • Provide useful information using standards (RDF, OWL, SPARQL) when URIs are looked up
  • Include links to other URIs so that they can discover more things

Resource Description Framework (RDF)

📋 RDF Overview

Resource Description Framework (RDF) is a W3C standard framework for representing information on the Web, developed to enable machine-readable data representation.

✨ Key Features

  • Framework to represent information on the Web
  • Graph formalism for describing interconnected entities
  • Highly versatile for describing digital objects, real-world objects, and concepts

🔺 The RDF Data Model

Triple Structure: The fundamental unit is a triple consisting of:

Subject

The resource being described

Predicate

The relationship/property

Object

The value or related resource

Subject → Predicate → Object Tim Berners-Lee → loves → Semantic Web

🔗 RDF Nodes

Three types of nodes:

1. IRIs (International Resource Identifiers)

  • Extend URIs to support international characters
  • Used for subjects and predicates
  • Example: http://dbpedia.org/page/Tim_Berners-Lee

2. Literals

Values with up to 3 parts:

  • Lexical form: Unicode string in quotes
  • Datatype IRI: determines how lexical form maps to value
  • Language tag: for language-specific strings
"1.01"^^xsd:decimal "Maastricht University"@en "Universiteit Maastricht"@nl

3. Blank Nodes

Nodes without persistent identifiers. Generally considered bad practice - assign IRIs instead.

🌐 URI Structure and Namespaces

URI Components

scheme:[//authority]path[?query][#fragment] https://springer.com/gp/book#9783030123741

Namespace Prefixes

@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> @prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#>

🏷️ RDF Predicates

Types

Data Properties

Object is a literal value

Object Properties

Object is an IRI

Built-in Predicates

  • rdf:type - specifies class membership ("is a" relation)
  • rdfs:label - provides human-readable labels

External Predicates

From vocabularies like Schema.org. Browse at: https://lov.linkeddata.es/

💻 RDF Syntaxes

1. N-Triples

<http://example.org/RDF> <http://www.w3.org/2000/01/rdf-schema#label> "Resource Description Framework"@en .

2. Turtle (Terse RDF Triple Language)

@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> @prefix ex: <http://example.org/> ex:RDF rdfs:label "Resource Description Framework"@en .

3. RDF/XML

<?xml version="1.0"?> <rdf:RDF xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#" xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#"> <rdf:Description rdf:about="http://example.org/RDF"> <rdfs:label xml:lang="en">Resource Description Framework</rdfs:label> </rdf:Description> </rdf:RDF>

4. JSON-LD

{ "@context": { "ex": "http://example.org/", "rdfs": "http://www.w3.org/2000/01/rdf-schema#" }, "@id": "ex:RDF", "rdfs:label": "Resource Description Framework" }

⚡ Turtle Shortcuts

Multiple values for one property

<http://maastrichtuniversity.nl> ex:location ex:Maastricht, ex:Netherlands .

Multiple properties for one resource

UM: rdfs:label "Maastricht University"; rdf:type ex:University; ex:location ex:Maastricht, ex:Netherlands.

📊 RDF Graphs

An RDF Graph is a labeled, directed multigraph where:

  • Nodes are subjects and objects
  • Edges are predicates
  • Multiple triples can share nodes, creating connections

Named Graphs

  • RDF graphs can be identified by URIs
  • Serialized using TriG, N-Quads, or JSON-LD formats
